[rivet]

cargo for C++. one binary, one toolchain, one toml. ships its own clang, drives vcpkg as a backend, builds itself.

install — detecting… 
█

⚡

Bundled LLVM

One rivet toolchain install drops clang-19, lld, libc++ on disk. No system compiler. No apt install build-essential.

⌬

vcpkg as a backend

rivet add fmt works on every platform. Per-project install trees. Auto-link through pkg-config. You never type vcpkg.

⌃

Multi-target manifests

[[lib]] / [[bin]] / [[test]] / [[vendor]] with cargo-style cfg(os="…") overrides and per-source flags.

∞

Self-hosting

Every release of rivet is built by rivet from its own rivet.toml. 49 sources, 5 artefacts, no cmake in the trace.

⎈

Three platforms

Linux x86_64 + arm64, macOS arm64, Windows x86_64. All four runners pass the nightly smoke end-to-end.

⌖

Hermetic

Content-addressed compile cache, in-process libzstd, no host tar/zstd prereq, no shared global vcpkg root.

$ quick start

01
rivet new hello-fmt && cd hello-fmt

Scaffold a new project. src/main.cpp, rivet.toml, .gitignore.

02
rivet add fmt

Register fmt as a dependency. Resolved through vcpkg under the hood.

03
rivet fetch

vcpkg builds fmt with rivet's bundled clang into ~/.rivet/cache/deps/.

04
rivet build && rivet run

Compile, link, execute. Prints hello from fmt 42.
